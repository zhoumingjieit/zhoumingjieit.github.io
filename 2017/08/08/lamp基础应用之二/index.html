lamp 详解

php嵌入式web应用编程语言

★ 作者：
    Rasmus Lerdorf,Personal Home Page tool, C语言（CGI，Web Forms）, PHP/FI
    zeev, andi重写php的解析器，
        zend engine 1.0, php 4.0;

★ zend engine引擎：
☉分两各个阶段：解析和执行
    解析：编译 –> opcode操作码
☉整个过程分为四步
    1）Scanning(Lexing)：扫描，做词法分析
    2）Parsing：语法分析
    3）Compilation：编译成二进制格式
    4）Excution：执行
前三步为解析编译过程，最后为执行过程。
☉hhvm：
    将PHP代码转换成高级别的字节码（通常称为中间语言）。然后在运行时通过即时（JIT）编译器将这些字节码转换为x64的机器码；
★ php加速器：
    基于php的特殊扩展机制，如opcode缓存扩展可以将opcode缓存于php的共享内存中，从而可以让同一段代码的后续重复执行时跳过编译阶段以提高性能。由此，可以看出，这些加速器并非真正提高了opcode的运行速度，而仅是通过分析opcode后并将他们重新排列以达到快速执行的目的。
☉常见的php加速器：
    APC，eAccelerator
    XCache：快速而且稳定的php opcode缓存，经过严格测试被大量应用于生产环境（会提升2~3倍）；

FastCGI之php-fpm

★ FastCGI：php以fpm机制独立地监听在一个套接字上；
    CGI全称“通用网关接口”（Common Gateway Interface）,用于HTTP服务器与其它主机上的程序服务通信交流的一种工具，CGI程序须运行在网络服务器上。传统CGI接口方式的主要缺点是性能较差。因为每次HTTP服务器遇到动态程序时都需要重新启动解析器来执行解析。然后结果被返回给HTTP服务器。这在处理高并发访问时几乎是不可用的，因此就诞生了FastCGI,另外传统的CGI接口方式安全性也很差，现在已经很少被用了。
    FastCGI是一个可伸缩地、高速地在运行HTTP服务器和动态脚本语言间通信的接口（FastCGI接口在Linux下是一个socket）主要优点是把动态语言和HTTP服务器凤离开来。FastCGI接口方式采用C/S架构，可将HTTP服务器和脚本解析服务器分开，同时在脚本解析服务器上启动一个或者多个脚本解析守护进程。当HTTP服务器每次遇到动态程序时，可以将其直接交付给FastCGI进程来执行，然后将得到的结果返回给浏览器，这种方式可以主HTTP服务器专一地处理静态请求或者将动态脚本服务器的结果返回给客户端，这在很大程序上提高了整个应用系统的性能。
★ php-fpm

☉安装方式：

注意：
    php-fpm和php二者不能同时存在会起冲突的，只能选择其一
☉主程序
    /usr/sbin/php-fpm
    Unit File（开机自启的服务）：/usr/lib/systemd/system/php-fpm.service
☉配置文件：
◆ini：配置php解释器工作环境；
    /etc/php.ini, /etc/php.d/.ini
◆conf：配置fpm守护进程的工作模式；
    /etc/php-fpm.conf, /etc/php-fpm.d/.conf
◆守护进程：
    守护进程（Daemon）是一种运行在后台的特殊进程，它独立于控制终端并且周期性的执行某种任务或等待处理某些发生的事件
★ 配置文件修改设置如下：
全局进程配置文件：/etc/php-fpm.conf
    [global]：定义全局进程自己的工作特性
连接池配置文件：/etc/php-fpm.d/www.conf
    [pool-id]：定义连接池，在包含的子文件/etc/php-fpm.d/www.conf中

案例：部署基于php-fpm的lamp环境

1.用yum安装php-fpm的软件包，安装之后编辑其配置文件，并根据配置文件要求创建/var/lib/php/session 目录，且属主和属组为apache，然后启动服务
php-fpm安装

编辑配置文件/etc/php-fpm.d/www.conf
[root@centos7 ~]# vim /etc/php-fpm.d/www.conf
[root@centos7 ~]# sed ‘/^;[[:space:]]\?.*/d’ /etc/php-fpm.d/www.conf
[www]
listen = 127.0.0.1:9000
listen.backlog = -1
listen.allowed_clients = 127.0.0.1
user = apache
group = apache
pm = dynamic
pm.max_children = 50
pm.start_servers = 5
pm.min_spare_servers = 5
pm.max_spare_servers = 35
pm.status_path = /php-status
ping.path = /ping
ping.response = pong
slowlog = /var/log/php-fpm/www-slow.log
rlimit_files = 1024
rlimit_core = 0
php_admin_value[error_log] = /var/log/php-fpm/www-error.log
php_admin_flag[log_errors] = on
php_value[session.save_handler] = files
php_value[session.save_path] = /var/lib/php/session

注意：
    httpd-2.2默认没有自带fcgi模块；php-5.3.3-之前的版本默认不支持fpm，需要打补丁方可；
    解决方案：编译安装httpd-2.4, php-5.3.3+；
    CentOS 6.X：以php-fpm方式部署lamp环境需要安装的程序包如下：
    yum install httpd-2.4 mod_proxy_fcgi php-fpm php-mysql mysqld-server

总结：

★ lamp环境的部署，主要是在httpd+php的方式上的不同
    一种是使用模块化的方式，要首先要判断httpd所使用的是什么MPM，如果使用的是prefork模块的话，则php所采用的模块为libphp，如果httpd使用的是worker或者event模块的话，则php采用就是libphp-zts模块；但是不管使用哪个模块，此时的httpd服务器相当于一个正向代理，即可加载动态资源又可加载静态资源；
    第二种是结合方式的FastCGI方式，即：php以fpm机制独立地监听在一个套接字上；这时，httpd是一个反向代理服务器，只处理客户端请求的静态资源，动态资源则反向代理至fcgi服务器，由后端服务器处理后返回结果。

    CentOS 6：
        Modules：程序包，httpd, php, php-mysql, mysql-server
