设备类型：

设备文件：
    b: 按块(block)为单位，随机访问的设备；（硬盘）
    c：按字符(character)为单位，线性设备；（键盘）
/dev下设备号：
    主设备号(major number):区分设备类型；用于标明设备所需要的驱动程序
    次设备号(minor number):区分同种类型下的不同的设备；是特定设备的访问入口
/dev/zero：
    是一个输入设备，你可你用它来初始化文件，该设备无穷尽地提供0(是ASCII 0)，熟称冒泡设备
/dev/null：
    该设备是空设备，也称为位桶（bit bucket）外号叫无底洞，任何写入它的输出都会被抛弃

磁盘分区：

主分区：
     一个硬盘只能创建4个主分区，为了创建多个分区：所以有了扩展分区
扩展分区：
    扩展分区里面就是一个指针指向整个磁盘中一个额外更换大的分区。
主分区和扩展分区的标识：1-4
逻辑分区：在扩展分区下创建的分区：
逻辑分区标识：5+
磁盘分区方式：
    主+扩展 <= 4 （扩展分区只能有一个）
注意：CentOS6及以前磁盘分区是按柱面划分的,CentOS7磁盘分区是按扇区划分的.

磁盘分区内部结构：

硬盘一个分区内部结构剖析：
一个分区划分成二个区域：
元数据存储区：
    inode条目：条目中都是每一个文件条目，每一个文件条目都有一个inode和对应的数据块位置
    inode: index node（inode号,对应文件权限，属主属组，时间戳，大小，对应文件存储的磁盘块.信息…）里面未存放文件名。文件名存放在哪里？（通过根目录一层一层根据inode号和对应磁盘块里的表信息向下找，最终找到文件名。每个磁盘块都有一个inode和文件名对应的信息表）
    inode位图：每个条目都对应一个位图，方便找空闲条目
    块位图：（每个存储块在这里面都有一个对应的位0或1），加速查找空闲块的机制。
数据存储区：
    逻辑存储单元（存储块）

一个分区划分成多个块组：
一个磁盘分区由于多个块组组成，每个块组上又都包括：元数据存储区，数据存储区.任何分区的第一个块为Boot Block。
超级块:（super block）一般都存储在第一个块组中
    保存整个分区的全局信息（有多少个块组，每个块组上包含多少个块，块大小，空闲磁盘块，已用磁盘块，空闲inode,已用inode…..）
块大小：block size一般为（1024byte,2048,4096）
块组描述符表：GDT,保存当前块组上的相关信息

磁盘分区类型：MBR、GPT
MBR：Master Boot Record，保存在0扇区(sector)
分为三部分：
    446bytes：bootloader, 程序，引导启动操作系统的程序；
    64bytes：分区表，每16bytes标识一个分区，一共只能有4个分区；
        4主分区
        3主1扩展、2主1扩展、1主1扩展：
            n逻辑分区
    2bytes：MBR区域的有效性标识；55AA为有效；

文件系统管理 ：

文件系统：
一个分区划分成二个区域：
元数据存储区：
    inode条目：条目中都是每一个文件条目，每一个文件条目都有一个inode号和对应的数据块位置
    index node(inode号,对应文件权限，属主属组，时间戳，大小，对应文件存储的磁盘块.信息…)，但里面未存放文件名。
    文件名存放在哪里？
    （通过根目录一层一层根据inode号和对应磁盘块里的表信息向下找，最终找到文件名.每个磁盘块上都有一个inode和文件名对应的信息表）
    位图索引：（每个条目都对应一个位图，方便找空闲条目）
    块位图：（每个存储块在这里面都有一个对应的位0或1），加速查找空闲块的机制。
数据存储区：
    逻辑存储单元（存储块）

新建一个文件过程：

    前提要有一个目录存在，首先扫描编目位图，找一个空闲条目（inode）,于是占用这个inode号，先找到目录对应inode,然后再找到这个inode对应的磁盘块，再在这个磁盘块中找到对应的表，在这个表上建立要新建的文件，然后把之前占用的inode号于这个新建的文件对应好存在这个表中。最后向文件中存数据，这时扫描块位图，找到空间块分配相应空间。

删除一个文件过程：

先找到文件在磁盘块中对应的表中的条目信息，删除文件条目，然后在编目位图中所这个条目对应的inode号标记空。然后找到对应的块，在块位图中把所对应的块标记为空。块里面的数据并为删除（只要这些块没有被再次使用，这些数据就存在）直到这些块被再次使用。

复制一个文件过程：速度慢

复制一个文件的过程和新建一个文件过程一样。

剪切一个文件过程

同一个分区剪切一个文件过程：速度快，剪切一个文件，如果在同一个文件系统（同一个分区）来说，只是把文件对应的路径移到其它目录下面，但inode还原来的inode,数据块还是原来的数据块。
跨分区剪切一个文件过程：速度慢，跨分区会先把文件复制到另一个分区上，然后删除原分区上的相关信息，所以会非常慢。

VFS: Virtual File System

    是内核中的一种文件系统，它将不同的文件系统统一起来为用户提供文件系统功能，用户管理文件系统时
只要访问VFS文件系统，然后VFS文件系统再调用内核功能去完成相关功能。
    Linux的文件系统: ext2(无日志功能), ext3, ext4, xfs, reiserfs, btrfs
    光盘：iso9660
    网络文件系统：nfs, cifs
    集群文件系统：gfs2, ocfs2
    内核级分布式文件系统：ceph
    windows的文件系统：vfat, ntfs
    伪文件系统：proc, sysfs, tmpfs, hugepagefs
    Unix的文件系统：UFS， FFS， JFS
    交换文件系统：swap
    用户空间的分布式文件系统：mogilefs, moosefs, glusterfs

日志文件系统：

    journal file system（加快文件系统修复速度），先把inode区存放到日志文件系统区，等数据块存放完后再把inode区放回到元数据区

链接文件：硬链接和符号链接

硬链接：
    直接指向同一个inode的不同路径
    删除文件其中一个硬链接只是把这个硬链接对应inode号的这条路径删除了，对inode号和文件内容没有影响。
    只能对文件创建，不能应用于目录
    不能跨文件系统
    创建硬链接会增加文件被链接的次数
创建硬链接：
    ln src link_file

符号链接：
    符号链接指向的是另一个文件的路径，大小等于指向文件的字符长度，访问文件的权限和符号链接的权限无关，
    所以符号链接的权限都是777。权限和指向的文件权限有关。
    是两个各自独立的文件，各自有自己的inode号。
    能应用于目录
    可以跨文件系统
    创建硬链接不会增加文件被链接的次数
创建符号链接：
    ln -s src link_file
    -v 显示创建过程

文件系统的使用：

mount命令和umount命令：

根文件系统这外的其它文件系统要想能够被访问，都必须通过“关联”至根文件系统上的某个目录来实现，此关联操作即为“挂载”；此目录即为“挂载点”；

挂载点：mount_point，用于作为另一个文件系统的访问入口；
    (1) 事先存在；
    (2) 应该使用未被或不会被其它进程使用到的目录；
    (3) 挂载点下原有的文件将会被隐藏；

mount命令：

命令格式：

命令选项：
    -r：readonly，只读挂载；

    -n：默认情况下，设备挂载或卸载的操作会同步更新至/etc/mtab文件中；-n用于禁止此特性；
    -t vfstype：指明要挂载的设备上的文件系统的类型；多数情况下可省略，此时mount会通过blkid来判断要挂载的设备的文件系统类型；

    -U UUID：挂载时以UUID的方式指明设备；

-o options：挂载选项
    sync/async：同步/异步操作；
    atime/noatime：文件或目录在被访问时是否更新其访问时间戳；

    remount：重新挂载；

    如：mount -o acl device dir

每行定义一个要挂载的文件系统及相关属性：分别都对应以下6个字段
    (1) 要挂载的设备：
        设备文件；
        LABEL
        UUID
        伪文件系统：如sysfs, proc, tmpfs等
    (2) 挂载点
        swap类型的设备的挂载点为swap；
    (3) 文件系统类型；
    (4) 挂载选项
        defaults：使用默认挂载选项；
        如果要同时指明多个挂载选项，彼此间以事情分隔；(defaults,acl,noatime,noexec)
    (5) 转储频率
        0：从不备份；
        1：每天备份；
        2：每隔一天备份；
    (6) 自检次序
        0：不自检；
        1：首先自检，通常只能是根文件系统可用1；
        2：次级自检
