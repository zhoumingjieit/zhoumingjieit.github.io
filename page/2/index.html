Linux网络接口命名方式

传统命名方式：

    以太网：ethX,例如：eth0,eth1

可预测命令方式(CentOS)：支持多种不同的命名机制：Firmware, 拓扑结构

    (1) 如果Firmware或BIOS为主板上集成的设备提供的索引信息可用，则根据此索引进行命名
     例如:eno1, eno2, …(eno16777736,eno33554984)
    (2) 如果Firmware或BIOS为PCI-E扩展槽所提供的索引信息可用，且可预测，则根据此索引进行命名
     例如ens1, ens2, …
    (3) 如果硬件接口的物理位置信息可用，则根据此信息命名，
     例如enp2s0, …
    (4) 如果用户显式定义，也可根据MAC地址命名，
     例如enx122161ab2e10, …

命令格式的组成说明：

    en：ethernet
    wl：wlan
    ww：wwan

名称类型：

    o：集成设备的设备索引号；
    s：扩展槽的索引号；
    x：基于MAC地址的命名；
    ps：基于总线及槽的拓扑结构进行命名；

程序包编译安装介绍：

rpm源码包：
如：testapp-VERSION-release.src.rpm
    需要安装后，使用rpmbuild命令制作成二进制格式的rpm包，而后再安装

C代码编译安装三步骤：

./configure：
    (1) 通过选项传递参数，指定启用特性、安装路径等；执行时会参考用户的指定以及Makefile.in文件生成makefile；
    (2) 检查依赖到的外部环境；
make：
    是用来编译的，它从Makefile中读取指令，然后编译，构建应用程序；
make install：
    是用来安装的，拷贝make编译好的文件安装到指定的位置
编译过程需要开发工具：
    autoconf: 生成configure脚本
    automake：生成Makefile.in
建议：安装前查看INSTALL，README

编译安装：

第一步：configure脚本

选项：指定安装位置、指定启用的特性
    –help: 获取其支持使用的选项
选项分类：
Installation directories: 安装路径设定：
    –prefix=/PATH/TO/SOMEWHERE: 指定默认安装位置；默认为/usr/local/
    –sysconfdir=/PATH/TO/SOMEWHERE：配置文件安装位置；
System types: 系统选项
Optional Features：可选特性
    –disable-FEATURE
    –enable-FEATURE[=ARG]
Optional Packages: 可选依赖包
    –with-PACKAGE[=ARG]
    –without-PACKAGE

第二步：make
第三步：make install

如果不是默认安装路径需要编译后配置（根据自己情况）

Linux程序包管理之yum仓库管理功能全解

2017-06-02

什么是YUM

    yum是redhat及其衍生版本中程序包管理器，主要是为了解决rpm包在安装过程中的依赖性问题而开发出来的一种前端管理工具。

yum程序包管理工具可分为服务端和客户端：

yum服务器

yum repository（yum repo）：存储了众多rpm包，以及包的相关的元数据文件（放置于特定目录下：repodata）
所有要发行的rpm包都放在yum服务器上以提供别人来下载，rpm包根据kernel的版本号，cpu的版本号分别编译发布。yum服务器只要提供简单的下载就可以了，ftp或者httpd的形式都可以。yum服务器有一个最重要的环节就是整理出每个rpm包的基本信息，包括rpm包对应的版本号，conf文件，binary信息，以及很关键的依赖信息。在yum服务器上提供了createrepo工具，用于把rpm包的基本概要信息做成一张”清单”，这张”清单””就是描述每个rpm包的spec文件中信息

yum clent端

client每次调用yum install或者search的时候，都会去解析/etc/yum.repos.d下面所有以.repo结尾的配置文件，这些配置文件指定了yum服务器的地址。yum会定期去”更新”yum服务器上的rpm包”清单”，然后把”清单”下载保存到yum自己的cache里面，根据/etc/yum.conf里配置(默认是在/var/cache/yum下面)，每次调用yum装包的时候都会去这个cache目录下去找”清单”，根据”清单”里的rpm包描述从而来确定安装包的名字，版本号，所需要的依赖包等，然后再去yum服务器下载rpm包安装。(前提是不存在rpm包的cache)

yum仓库访问方式：

    ftp://
    http://
    nfs://
    file:///

yum配置文件

    /etc/yum.conf：为所有仓库提供公共配置
    /etc/yum.repos.d/*.repo：为仓库的指向提供配置

全局配置文件：/etc/yum.conf

    [root@centos7 ~]# cat /etc/yum.conf
    [main]
    cachedir=/var/cache/yum/$basearch/$releasever –>定义缓存目录
    keepcache=0 –>是否需要在本地缓存程序包
    debuglevel=2 –>调试级别,运行过程中的信息是否需要显示
    logfile=/var/log/yum.log –>日志文件,记录yum命令安装过的文件,执行过的操作等等,供以后检索使用
    exactarch=1 –>下载的软件包是否需要和本地平台完全匹配（1表示是，0表示否）
    obsoletes=1
    gpgcheck=1 –>下载rpm包之前是否需要自动进行来源（签名）合法性检测,1表示要检查
    plugins=1
    installonly_limit=5
    bugtracker_url=http://bugs.centos.org/set_project.php?project_id=23&ref=http://bugs.centos.org/bug_report_page.php?category=yum
    distroverpkg=centos-release

仓库指向的定义：/etc/yum.repo.d/*.repo

    [repositoryID] –>仓库的ID,可以取任意名字,只要不和其他的ID冲突即可
    name=Some name for this repository –>仓库的描述信息,长短不限,可以有空格,但是必不可少
    baseurl=url://path/to/repository/ –>告诉yum客户端,通过哪种路径可以访问到
    enabled={1|0} –>是否启用这个仓库,0表示不启用,1表示启用，默认是启用的
    gpgcheck={1|0} –>是否进行签名合法性检测,0表示不启用,1表示启用,默认启用
    gpgkey=URL –>如果启用gpg检测,则需要指定gpgkey的路径,需要手动指定路径,这个路径可以是远程服务器上
    enablegroups={1|0}
    failovermethod={roundrobin|priority} –>默认为：roundrobin，意为随机挑选；
    cost= –>设置优先级,默认为1000 访问的代价，定义了cost的,且数字较小的,会被优先访问：

linux系统自带仓库指向定义配置文件中可用变量：

    [base]
    name=CentOS-$releasever - Base
    mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=os&infra=$infra

    gpgcheck=1
    gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-CentOS-7
    变量说明：
    $releasever: 当前OS的发行版的主版本号；
    $arch: 平台；
    $basearch：基础平台；
    $YUM0-$YUM9

磁盘分类：

    SATA:

机械磁盘：

盘片：platters
轴：spindle
读写头：read/write head
扇区：sector（512字节）
磁道：track
柱面：cylinder
寻道时间：磁头切换柱面寻找数据时所花的时间。磁盘转的越快，寻道时间越短，磁盘性能越好。

设备类型：

设备文件：
    b: 按块(block)为单位，随机访问的设备；（硬盘）
    c：按字符(character)为单位，线性设备；（键盘）
/dev下设备号：
    主设备号(major number):区分设备类型；用于标明设备所需要的驱动程序
    次设备号(minor number):区分同种类型下的不同的设备；是特定设备的访问入口
/dev/zero：
    是一个输入设备，你可你用它来初始化文件，该设备无穷尽地提供0(是ASCII 0)，熟称冒泡设备
/dev/null：
    该设备是空设备，也称为位桶（bit bucket）外号叫无底洞，任何写入它的输出都会被抛弃
创建设备文件：

文件系统管理 ：

文件系统：
一个分区划分成二个区域：
元数据存储区：
    inode条目：条目中都是每一个文件条目，每一个文件条目都有一个inode号和对应的数据块位置

    逻辑存储单元（存储块）

新建一个文件过程：

    前提要有一个目录存在，首先扫描编目位图，找一个空闲条目（inode）,于是占用这个inode号，先找到目录对应inode,然后再找到这个inode对应的磁盘块，再在这个磁盘块中找到对应的表，在这个表上建立要新建的文件，然后把之前占用的inode号于这个新建的文件对应好存在这个表中。最后向文件中存数据，这时扫描块位图，找到空间块分配相应空间。

删除一个文件过程：

先找到文件在磁盘块中对应的表中的条目信息，删除文件条目，然后在编目位图中所这个条目对应的inode号标记空。然后找到对应的块，在块位图中把所对应的块标记为空。块里面的数据并为删除（只要这些块没有被再次使用，这些数据就存在）直到这些块被再次使用。

复制一个文件过程：速度慢

复制一个文件的过程和新建一个文件过程一样。

剪切一个文件过程

同一个分区剪切一个文件过程：速度快，剪切一个文件，如果在同一个文件系统（同一个分区）来说，只是把文件对应的路径移到其它目录下面，但inode还原来的inode,数据块还是原来的数据块。
跨分区剪切一个文件过程：速度慢，跨分区会先把文件复制到另一个分区上，然后删除原分区上的相关信息，所以会非常慢。

链接文件：硬链接和符号链接

硬链接：
    直接指向同一个inode的不同路径
    删除文件其中一个硬链接只是把这个硬链接对应inode号的这条路径删除了，对inode号和文件内容没有影响。
    只能对文件创建，不能应用于目录
    不能跨文件系统
    创建硬链接会增加文件被链接的次数
创建硬链接：
    ln src link_file

符号链接：
    符号链接指向的是另一个文件的路径，大小等于指向文件的字符长度，访问文件的权限和符号链接的权限无关，
    所以符号链接的权限都是777。权限和指向的文件权限有关。
    是两个各自独立的文件，各自有自己的inode号。
    能应用于目录
    可以跨文件系统
    创建硬链接不会增加文件被链接的次数
创建符号链接：
    ln -s src link_file
    -v 显示创建过程

API：Application Progran Interface

    应用程序接口（英语：Application Programming Interface，简称：API），又称为应用编程接口，就是软件系

统不同组成部分衔接的约定。由於近年來软件的规模日益庞大，常常需要把复杂的系统划分成小的组成部分，编程接口

的设计十分重要。程序设计的实践中，编程接口的设计首先要使软件系统的职责得到合理划分。良好的接口设计可以

降低系统各部分的相互依赖，提高组成单元的内聚性，降低组成单元间的耦合程度，从而提高系统的维护性和扩展性。

    其主要目的是讓應用程式開發人員得以呼叫一組例程功能，而無須考慮其底層的原始碼為何、或理解其内部工作

機制的細節。API本身是抽象的，它僅定義了一個介面，而不涉及应用程序在实际实现过程中的具体操作。

ABI：Application Binary Interface

    应用二进制接口（英语：application binary interface，縮寫為 ABI）描述了应用程序（或者其他类型）

和操作系统之间或其他应用程序的低级接口。

    ABI不同于应用程序接口（API），API定义了源代码和库之间的接口，因此同样的代码可以在支持这个API的

任何系统中编译，然而ABI允许编译好的目标代码在使用兼容ABI的系统中无需改动就能运行。

时间表示方法:
(1) 特定值：

     给定时间点有效取值范围内的值（注意：day of week和day of month一般不同时使用）

(2) *

     给定时间点上有效取值范围内的所有值；表"每...""

(3) 离散取值：,(逗号)

     在时间点上使用逗号分隔的多个值

     如：#,#,#

(4) 连续取值：-

     在时间点上使用-连接开头和结束

     如：#-#

(5) 在指定时间点上,定义步长；表示"每..."

     如：*/# 这里的#表示步长
示例
(1) 每小时间执行一次：每小时的第3分钟

    3    *    *    *    *

(2) 每周执行一次：每周5的4点3分

    3    4    *    *    5

(3) 每月执行一次：每月的7号6点5分

    5    6    7    *    *

(4) 每年执行一次：每年的10月9号8点7分

    7    8    9    10    *

(5) 每周三和周日的8点9分执行：

    9    8    *    *    3,7

(6) 每周三和周日的早8点和晚8点整执行：

    0    8,20    *    *    3,7

(7) 每周一到周五的上午9点到下午18点执行：

    0    9-18    *    *    1-5

(8) 每5分钟执行一次：

    */5    *    *    *    *
